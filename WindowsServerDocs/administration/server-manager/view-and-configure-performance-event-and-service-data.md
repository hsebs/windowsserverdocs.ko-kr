# <a name="view-and-configure-performance-event-and-service-data"></a>성능, 이벤트 및 서비스 데이터 보기 및 구성

>적용 대상: Windows Server(반기 채널), Windows Server 2016, Windows Server 2012 R2, Windows Server 2012

이 항목에서는 이벤트 로그 항목, 성능 카운터 및 서버 관리자에서 로컬 및 원격 서버에 대해 표시 되는 서비스 경고 보기 및 구성 하는 방법을 설명 합니다.  

이벤트, 서비스 및 성능 로그 데이터는 Windows Server의 서버 관리자 콘솔의 두 위치에 표시 됩니다.  

-   대시보드, 클릭 수는 **이벤트**, **성능**, 및 **서비스** 의 축소판 그림 이벤트, 성능 및 역할, 전체 서버 관리자 서버 풀, 서버, 사용자가 만든 그룹 및 로컬 서버에 대 한 확인 하려는 서비스 로그 데이터를 구성 하는 행입니다. 하이퍼텍스트 행을 클릭 하면 **자세히 보기** 대화 상자가 열리며,이 대화 상자에서 대시보드에 경고를 표시 하려는 데이터를 지정할 수 있습니다. 대시보드 미리 보기에 강조 표시할 이벤트, 서비스 및 성능 로그 데이터를 구성한 후에는 사용자가 지정한 조건과 일치 하는 로그 항목이 **자세히 보기** 대화 상자 아래쪽에 나열 됩니다.  

-   **이벤트**, **서비스** 및 **성능** 타일은 역할과 그룹 홈 페이지의 일부입니다. 이러한 타일의 **작업** 메뉴에 있는 명령을 사용하여 관리 서버에서 수집하려는 데이터를 지정할 수 있습니다. 타일에는 원하는 경우 타일에 표시되는 로그 항목을 더 제한할 수 있는 필터와 쿼리가 포함되어 있습니다.

## <a name="what-are-thumbnails"></a><a name=BKMK_thumb></a>축소판 그림 이란?  
*축소판 그림* (역할의 미리 보기는 모든 역할을 실행 하는 서버 관리자 풀의 서버에 대 한 수집 된 데이터를 반영 하는 데 사용) 각 역할에 대해 서버 관리자 대시보드에 표시 된 각 서버 그룹에 대 한에 대 한는 **모든 서버** (서버 관리자 서버 풀에 있는 모든), 그룹 및 로컬 서버에 대 한 합니다. 서버 관리자는 관리 되는 서버에서 데이터를 가져오기, 미리 보기에서 서버 풀의 서버에서 실행 되는 역할에 대 한 자동으로 생성 됩니다.  

서버 관리자 콘솔이 원격 서버 관리 도구의 일부로 클라이언트 컴퓨터에서 실행 되는 경우 **로컬 서버** 미리 보기는 없습니다.  

미리 보기에는 역할, 서버 및 서버 그룹의 상태와 관리 효율성이 간략하게 표시됩니다. 이벤트, 성능 카운터, 모범 사례 분석기 결과, 서비스 또는 일반적인 관리 효율성 문제가 **자세히 보기** 대화 상자에서 구성 하는 조건을 충족 하는 경우 (미리 보기 행을 클릭 하 여 연) 축소판 그림 머리글 행의 색이 변경 되 고 강조 표시 된 숫자가 왼쪽 여백에 표시 됩니다. 다음 표에서는 미리 보기에 표시되는 데이터에 대해 설명합니다.  

|미리 보기 행|설명|  
|---------|--------|  
|관리 효율성|서버의 관리 효율성에는 여러 가지 측정값이 포함 됩니다: 있는지 여부를 서버 온라인 또는 오프 라인으로 액세스할 수는 것 이며 원격으로 관리 해야 하는 원격 서버에서의 실행 여부는 소프트웨어를 모두 있는 보고 데이터 서버 관리자를 로컬 컴퓨터에 로그온 된 사용자에 액세스 하거나 원격 서버를 관리할 수 있는 사용자 권한이 있는지 여부 또는 서버를 쿼리하고 서버 관리자를 사용 하 여 관리 하도록 허용 하는 방식으로 구성 되어 있는지 여부. Windows Server 2003을 실행 하는 서버에서 서버 관리자를 수집할 수 있는 유일한 관리 효율성 데이터는 온라인 또는 오프 라인 서버 인지 합니다. 관리 효율성 상태 오류 및 해결 방법에 대한 자세한 내용은 [서버 관리자 문제 해결 가이드](https://social.technet.microsoft.com/wiki/contents/articles/13443.windows-server-2012-server-manager-troubleshooting-guide-part-i-overview.aspx)를 참조하세요.|  
|이벤트|지정한 심각도 수준, 원본, 기간, 서버 또는 이벤트 ID와 일치하는 이벤트가 기록될 때 경고를 표시하도록 미리 보기의 **이벤트** 행을 구성할 수 있습니다. 이벤트에 대 한 세부 정보를 확인 하 고 **이벤트** 행을 클릭 한 다음 역할 또는 서버 그룹의 **이벤트 자세히 보기** 대화 상자를 열어 보려는 경고를 변경 합니다.|  
|서비스|서비스 **자세히 보기** 대화 상자에서 지정한 시작 유형, 서비스 상태, 서비스 이름 및 서버와 일치 하는 서비스를 역할 또는 서버 그룹에서 찾으면 경고를 표시 하도록 **서비스** 행을 구성할 수 있습니다.<p>서버 관리자 서버 풀에 서버를 추가한 후에 관리 되는 서버에 로그온 사용자가 없을 경우 셸 하드웨어 검색 서비스에 대 한 서비스 알림은 표시할 수 있습니다. 이는 사용자가 관리 서버에 로그온되어 있거나 관리 서버의 원격 데스크톱 세션에 연결되어 있는 경우에만 셸 하드웨어 검색 서비스가 실행되기 때문입니다. 이 경우 셸 하드웨어 검색 서비스 경고가 표시되지 않도록 하려면 **모든 서버** 그룹이 포함된 서버 그룹의 미리 보기에서 **서비스** 를 클릭합니다. **서비스 자세히 보기** 대화 상자의 **서비스** 드롭다운 목록에서 **셸 하드웨어 검색**확인란의 선택을 취소 한 다음 **확인**을 클릭 합니다.|  
|성능|성능 **자세히 보기** 대화 상자에서 지정한 리소스 유형, 서버 또는 시간과 일치 하는 성능 경고가 발생 하면 역할 또는 서버 그룹에 대 한 경고를 표시 하도록 **성능** 행을 구성할 수 있습니다.<p>기본적으로 성능 카운터는 꺼져 있습니다. Windows Server 2003 보다 최신 버전의 운영 체제를 실행 하는 관리 서버 및 성능 카운터가 시작 되지 않은 경우 일반적으로 역할 또는 그룹 페이지의 **서버** 타일에서 **시작 되지 않은 온라인 성능 카운터** 의 관리 효율성 상태 오류가 표시 됩니다. 관리 되는 서버에 대 한 성능 카운터를 켜려면 **모든 서버** 페이지에서 **카운터 상태** 값이 **꺼짐**인 **성능** 타일의 항목을 마우스 오른쪽 단추로 클릭 한 다음 **성능 카운터 시작**을 클릭 합니다. 역할 또는 그룹 페이지의 **서버** 타일에서 서버 항목을 마우스 오른쪽 단추로 클릭 한 다음 **성능 카운터 시작**을 클릭 하 여 성능 카운터를 시작할 수도 있습니다.|  
|BPA 결과|Bpa **결과 자세히 보기** 대화 상자에서 지정한 심각도 수준, 서버 또는 bpa 범주와 일치 하는 bpa 검사 결과가 있으면 역할 또는 서버 그룹에 대 한 경고를 표시 하도록 **bpa 결과** 행을 구성할 수 있습니다.|  

## <a name="view-and-configure-events"></a><a name=BKMK_events></a>이벤트 보기 및 구성  
이 섹션에서는 서버 관리자 서버 풀의 서버에서 수집 되는 이벤트 로그 데이터를 구성 하는 방법을 알아보고 미리 보기에 강조 표시할 이벤트를 합니다.  

> [!NOTE]  
> 이벤트에 대 한 경고가 표시 미리 보기에는 관리 되는 서버에서 수집 하는 서버 관리자에 게 전체 이벤트의 하위 집합입니다. 있는 이벤트 조건을 변경 하지만 **이벤트 데이터 구성** 대화 상자에 **이벤트** 타일에서 관리 되는 서버에서 수집 되는 이벤트 로그 데이터에 영향을 주지 미리 보기에 있는 이벤트 경고 조건을 변경 합니다. 서버 관리자 대시보드의 참조 경고 번호를 변경할 수 있습니다.  

#### <a name="to-configure-the-events-collected-from-managed-servers"></a>관리 서버에서 수집된 이벤트를 구성하려면  

1.  서버 관리자 콘솔에서 대시보드를 제외한 모든 페이지를 엽니다. 역할, 서버 그룹 또는 로컬 서버 페이지의 **이벤트** 타일에서 관리 서버에서 수집하려는 이벤트를 구성할 수 있습니다.  

2.  **이벤트** 타일의 **작업** 메뉴에서 **이벤트 데이터 구성**을 클릭합니다.  

3.  선택한 그룹의 서버에서 수집 하려는 이벤트 심각도 수준을 선택 합니다. 기본적으로 **위험**, **오류** 및 **경고** 심각도 수준이 선택됩니다.  

4.  이벤트가 발생되는 시간을 지정합니다. 기본 이벤트 시간은 24시간입니다.  

5.  이벤트를 수집 하려는 이벤트 로그 파일을 선택 합니다. 기본 파일은 **응용 프로그램**, **설치**및 **시스템**입니다.  

6.  변경 사항을 저장하려면 **확인** 을 클릭하여 **이벤트 데이터 구성** 대화 상자를 닫습니다. 변경 사항이 저장되면 데이터가 자동으로 새로 고쳐집니다.  

#### <a name="to-configure-the-events-highlighted-in-thumbnails"></a>미리 보기에 강조 표시되는 이벤트를 구성하려면  

1.  서버 관리자 이미 열려 있는 경우 다음 단계로 이동 합니다. 서버 관리자가 아직 열려 있지 않으면 다음 중 하나를 수행하여 엽니다.  

    -   Windows 바탕 화면에서 Windows 작업 표시줄의 **서버 관리자**를 클릭하여 서버 관리자를 시작합니다.  

    -   Windows **시작** 화면에서 서버 관리자 타일을 클릭합니다.  

2.  대시보드 페이지에서 **역할 및 서버 그룹** 타일의 미리 보기에 있는 **이벤트** 행을 클릭합니다.  

3.  **이벤트 자세히 보기** 대화 상자에서 표시 하려는 이벤트의 심각도 수준을 추가 합니다. 기본적으로 미리 보기에는 위험 이벤트를 강조하는 경고만 표시됩니다. 경고를 발생 시킬 심각도 수준을 추가 하면 **자세히 보기** 대화 상자에 표시 되는 이벤트 수가 늘어납니다.  

4.  **이벤트 원본** 필드에서는 경고를 발생시킬 이벤트 원본을 선택합니다. 기본값은 **All**입니다.  

5.  이 미리 보기가 여러 서버 또는 여러 서버에 설치 된 역할에 대 한 미리 보기인 경우에는 **서버** 드롭다운 목록에서 이벤트 경고를 발생 시킬 서버를 선택할 수 있습니다.  

6.  **기간 필드에** 최대 1440 분, 24 시간 또는 1 일의 기간을 지정 합니다.  

7.  **이벤트 ID** 필드에는 경고를 발생시킬 특정 이벤트의 ID 번호를 입력합니다. 대시( **-** )를 사용하여 이벤트 ID의 범위를 입력할 수 있으며, 특정 이벤트 앞에 대시를 사용하면 해당 이벤트 ID를 범위에서 제외할 수 있습니다. 예를 들어 **1,3,5-99,-76** 값은 이벤트 ID 1과 3, 그리고 이벤트 ID 76을 제외한 ID 5부터 99까지의 모든 이벤트에 대해 경고가 발생됨을 의미합니다.  

8.  경고가 표시될 조건을 변경하면 대화 상자 하단에 있는 결과 창에 표시되는 이벤트 경고의 수도 변경될 수 있습니다. 목록에서 항목을 선택 하 고 **경고 숨기기** 를 클릭 하 여 원본 미리 보기에 표시 되는 경고 개수에 영향을 주지 않도록 합니다. **Ctrl** 키를 누른 채 경고를 선택하면 한 번에 여러 개의 경고를 선택할 수 있습니다. 이벤트 경고 발생 조건에는 맞지만 표시할 필요가 없는 경고의 경우 숨길 수 있습니다.  

9. 숨겨진 경고를 목록에 다시 표시하려면 **모두 표시** 를 클릭합니다.  

10. **확인** 을 클릭 하 여 변경 내용을 저장 하 고 **자세히 보기** 대화 상자를 닫은 다음 원본 미리 보기에서 변경 된 이벤트 경고를 확인 합니다.  

## <a name="view-and-configure-performance-log-data"></a><a name=BKMK_perf></a>성능 로그 데이터 보기 및 구성  
이 섹션에서는 서버 관리자 서버 풀의 서버에서 수집 되는 성능 로그 데이터를 구성 하는 방법을 알아보고 축소판 그림에 강조 표시할 성능 카운터 경고가 발생 합니다.  

기본적으로 성능 카운터는 꺼져 있습니다. Windows Server 2003 보다 최신 버전의 운영 체제를 실행 하는 관리 서버 및 성능 카운터가 시작 되지 않은 경우 일반적으로 역할 또는 그룹 페이지의 **서버** 타일에서 **시작 되지 않은 온라인 성능 카운터** 의 관리 효율성 상태 오류가 표시 됩니다. 관리 되는 서버에 대 한 성능 카운터를 켜려면 **모든 서버** 페이지에서 **카운터 상태** 값이 **꺼짐**인 **성능** 타일의 항목을 마우스 오른쪽 단추로 클릭 한 다음 **성능 카운터 시작**을 클릭 합니다. 역할 또는 그룹 페이지의 **서버** 타일에서 서버 항목을 마우스 오른쪽 단추로 클릭 한 다음 **성능 카운터 시작**을 클릭 하 여 성능 카운터를 시작할 수도 있습니다.  

> [!NOTE]  
> 축소판 그림에서 보면 성능 알림은 관리 되는 서버에서 수집 하는 서버 관리자에 지시 하는 전체 성능 카운터 데이터의 하위 집합입니다. 성능 경고 조건을 변경할 수 있지만 **성능 경고 구성** 대화 상자에 **성능** 타일에서 관리 되는 서버에서 수집 되는 성능 로그 데이터에 영향을 주지 미리 보기에는 성능 경고 조건을 변경할 서버 관리자 대시보드의 참조 경고 번호를 변경할 수 있습니다.  
>   
> 이 때문에 미리 보기에 표시할 수 있는 성능 데이터의 최대 보존 기간은 **성능 경고 구성** 대화 상자에서 구성된 최대 그래프 표시 기간보다 클 수 없습니다. 예를 들어 **성능 경고 구성** 의 **그래프 표시 기간** 값이 **1 일**이면 서버 관리자 대시보드에서 연 **성능 자세히 보기** 대화 상자의 **기간 필드 최대값** 은 **1 일**, **24 시간**또는 **1440 분**이 될 수 있습니다.  

#### <a name="to-configure-the-performance-log-data-collected-from-managed-servers"></a>관리 서버에서 수집되는 성능 로그 데이터를 구성하려면  

1.  서버 관리자 콘솔에서 대시보드를 제외한 모든 페이지를 엽니다. 역할, 서버 그룹 또는 로컬 서버 페이지의 **성능** 타일에서 관리 서버로부터 수집할 성능 데이터를 구성할 수 있습니다.  

2.  관리 서버에서 성능 로그 데이터를 수집하려면 성능 카운터가 켜져 있어야 합니다. 성능 카운터가 꺼져 있는 경우 **성능** 타일 목록의 항목을 마우스 오른쪽 단추로 클릭 한 다음 **성능 카운터 시작**을 클릭 합니다. 데이터가 수집될 서버의 수와 사용 가능한 네트워크 대역폭에 따라 성능 카운터 데이터를 수집하는 데 시간이 다소 소요될 수 있습니다. **카운터 상태** 열에서 상태를 확인합니다.  

3.  **성능** 타일의 **작업** 메뉴에서 **성능 경고 구성**을 클릭합니다.  

4.  선택한 그룹의 서버 또는 선택 된 역할을 실행 하는 서버에 대해 서버 관리자에서 수집 하는 성능 카운터 경고를 원하는 CPU 사용량 비율을 지정 합니다. 기본값은 85%입니다.  

5.  서버에 사용 가능한 메모리가 몇 MB 남아 있을 때 성능 카운터 경고가 수집되도록 할지 지정합니다. 기본값은 2MB입니다.  

6.  선택한 페이지의 **성능** 타일에서 **CPU 사용량** 및 **사용 가능한 메모리** 리소스가 그래프로 표시되는 시간을 지정합니다. 기본값은 1일입니다. **저장**을 클릭합니다.  

    **저장** 을 클릭한 후 **성능**타일의 성능 경고 수와 그래프로 표시되는 시간 동안의 경고 매핑이 변경될 수 있습니다.  

    > [!NOTE]  
    > [동적 메모리](https://technet.microsoft.com/library/ff817651.aspx) 설정 된 가상 컴퓨터의 경우 성능 경고 임계값을 높이면 가양성 경고가 발생할 수 있습니다.  

7.  서버에서 수집되는 성능 경고 목록을 새로 고치려면 **작업** 메뉴에서 **새로 고침**을 클릭합니다.  

#### <a name="to-configure-the-performance-alerts-highlighted-in-thumbnails"></a>미리 보기에 강조 표시되는 성능 경고를 구성하려면  

1.  대시보드 페이지에서 **역할 및 서버 그룹** 타일의 미리 보기에 있는 **성능** 행을 클릭합니다.  

2.  **성능 자세히 보기** 대화 상자에서 **리소스 유형** 필드에 경고를 표시 하려는 리소스 성능 임계값에 대 한 확인란을 선택 하거나 선택 취소 합니다. 경고를 발생 시킬 리소스 성능 임계값을 추가 하면 **자세히 보기** 대화 상자에 표시 되는 성능 경고 수가 늘어날 수 있습니다.  

3.  이 미리 보기가 여러 서버 또는 여러 서버에 설치 된 역할에 대 한 미리 보기인 경우에는 **서버** 드롭다운 목록에서 성능 경고를 발생 시킬 서버를 선택할 수 있습니다.  

4.  **기간 필드에** 최대 1440 분, 24 시간 또는 1 일의 기간을 지정 합니다.  

5.  경고가 표시되는 조건을 변경하면 대화 상자 하단의 결과 창에 표시되는 경고의 수도 변경될 수 있습니다. 현재 시각 이전의 모든 경고를 숨겨 원본 미리 보기에 표시되는 경고 개수에 반영되지 않도록 하려면 **경고 숨기기** 를 클릭합니다.  

6.  숨겨진 경고를 목록에 다시 표시하려면 **모두 표시** 를 클릭합니다.  

7.  **확인** 을 클릭 하 여 변경 내용을 저장 하 고 **자세히 보기** 대화 상자를 닫은 다음 원본 미리 보기에서 변경 된 성능 경고를 확인 합니다.  

#### <a name="to-view-the-properties-of-performance-alerts"></a>성능 경고의 속성을 확인하려면  

1.  다음 중 하나를 수행합니다.  

    -   대시보드 페이지에서 **역할 및 서버 그룹** 타일의 미리 보기에 있는 **성능** 행을 클릭합니다.  

    -   역할이나 그룹 홈 페이지를 열고 역할 또는 그룹의 **성능** 타일을 찾습니다.  

2.  목록에서 속성을 확인하려는 성능 경고를 두 번 클릭합니다. 또는 성능 경고를 마우스 오른쪽 단추로 클릭한 다음 **속성 보기**를 클릭합니다.  

3.  **성능 경고 속성** 대화 상자에서 로그 항목을 선택하면 **프로세스** 영역의 항목과 관련된 프로세스 관련 정보를 볼 수 있습니다.  

4.  성능 경고 속성을 확인하면 대화 상자를 닫습니다.  

### <a name="analyze-performance-data-and-solve-problems"></a>성능 데이터 분석 및 문제 해결  
서버 관리자에서 볼 수 있는 성능 카운터 데이터를 분석 하 고 관리 서버의 성능 문제를 해결 하는 방법에 대 한 자세한 내용은 다음 리소스를 참조 하십시오.  

-   [성능 데이터 분석](https://go.microsoft.com/fwlink/?LinkId=239829)  

-   [성능 문제 해결](https://go.microsoft.com/fwlink/?LinkId=239831)  

Server Performance Advisor 3.0를 비롯 하 여 windows server 2012 이상 버전에서 사용할 수 있는 고급 성능 모니터링 및 분석 도구에 대 한 자세한 내용은 MSDN의 [성능](https://msdn.microsoft.com/windows/hardware/gg463374.aspx) 을 참조 하십시오.  

## <a name="manage-services-and-configure-service-alerts"></a><a name=BKMK_services></a>서비스 관리 및 서비스 경고 구성  
이 섹션에서는 시작, 중지, 다시 시작, 일시 중지, 또는에 표시 되는 서비스를 다시 시작 하는 방법에 알아봅니다는 **서비스** 역할 및 서버 그룹 페이지에서 서버 관리자에서 타일입니다. 또한 서버 관리자 대시보드의 미리 보기에 경고가 표시 되는 방법에 대 한 서비스를 구성할 수 있습니다.  

> [!NOTE]  
> 서비스, 서비스 종속성, 복구 옵션 또는 서버 관리자의 서비스 타일에서 다른 서비스 속성에 대 한 시작 유형을 변경할 수 없습니다. 서비스 상태 이외의 서비스 속성을 변경하려면 **서비스** 스냅인을 열어야 합니다. 열에 대 한 바로 가기는 **서비스** 스냅인은에서 사용할 수는 **도구** 서버 관리자 메뉴.  

#### <a name="to-start-stop-restart-pause-or-resume-a-service"></a>서비스를 시작, 중지, 다시 시작, 일시 중지 또는 재시작하려면  

1.  서버 관리자 콘솔에서 대시보드 (즉, 모든 역할이 나 그룹 홈 페이지)를 제외한 모든 페이지를 엽니다.  

2.  역할 또는 그룹의 **서비스** 타일에서 서비스를 마우스 오른쪽 단추로 클릭합니다.  

3.  상황에 맞는 메뉴가 표시되면 서비스에 대해 수행하려는 작업을 클릭합니다. 서비스가 중지된 경우 서비스를 시작하는 작업만 수행할 수 있습니다. 마찬가지로 서비스가 일시 중지된 경우에는 서비스를 재시작하는 작업만 수행할 수 있습니다.  

4.  서비스를 시작, 중지, 다시 시작, 일시 중지 또는 재시작하면 **서비스** 타일에서 해당 서비스의 **상태** 열의 값이 변경됩니다.  

#### <a name="to-configure-service-alerts-highlighted-in-thumbnails"></a>미리 보기에 강조 표시되는 서비스 경고를 구성하려면  

1.  대시보드 페이지에서 **역할 및 서버 그룹** 타일의 미리 보기에 있는 **서비스** 행을 클릭합니다.  

2.  **서비스 자세히 보기** 대화 상자에서 경고를 받으려는 서비스의 시작 유형을 선택 합니다. 기본적으로 **자동 (지연 된 시작)** 및 **자동** 이 선택 됩니다.  

3.  경고를 표시 하려는 서비스 상태를 선택 합니다. 기본적으로 **모두** 가 선택됩니다.  

4.  경고를 표시 하려는 서비스를 선택 합니다. 기본적으로 **모두** 가 선택됩니다.  

5.  서비스에 대 한 경고를 수신 하려는 역할이 나 그룹에 연결 된 서버를 선택 합니다. 기본적으로 **모두** 가 선택됩니다.  

6.  경고가 표시되는 조건을 변경하면 대화 상자 하단의 결과 창에 표시되는 경고의 수도 변경될 수 있습니다. 현재 시각 이전의 모든 경고를 숨겨 원본 미리 보기에 표시되는 경고 개수에 반영되지 않도록 하려면 **경고 숨기기** 를 클릭합니다.  

7.  숨겨진 경고를 목록에 다시 표시하려면 **모두 표시** 를 클릭합니다.  

8.  **확인** 을 클릭 하 여 변경 내용을 저장 하 고 **자세히 보기** 대화 상자를 닫은 다음 원본 미리 보기에서 변경 된 서비스 경고를 확인 합니다.  

## <a name="view-and-copy-event-service-or-performance-entries"></a><a name=BKMK_copy></a>이벤트, 서비스 또는 성능 항목 보기 및 복사  
역할 또는 그룹의 이벤트 및 **성능** 타일과 **자세히 보기** 대화 상자 **에서 이벤트,** 서비스 또는 성능 항목 속성을 복사할 수 있습니다. 이벤트 또는 성능 항목을 마우스 오른쪽 단추로 클릭 한 다음 **복사**를 클릭 합니다.  

**이벤트** 타일에서 목록에 있는 이벤트를 선택하면 타일의 중간 하단에서 이벤트 속성을 미리 볼 수 있습니다. 미리 보기에 표시 된 속성을 복사 하려면 미리 보기 창을 마우스 오른쪽 단추로 클릭 한 다음 **복사**를 클릭 합니다.
